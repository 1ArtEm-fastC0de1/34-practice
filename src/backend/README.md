# Backend – Practice Project

## 📌 Опис
Backend частина проєкту реалізована на **Node.js + Express**.  
Вона відповідає за обробку HTTP-запитів, роботу з моделями даних, аутентифікацію на основі **JWT**, керування кукі та взаємодію з базою даних. Архітектура побудована за модульним принципом: DTO → Middleware → Router → Service → Model.  
Проєкт контейнеризований за допомогою **Docker**, що дозволяє легко розгортати його у різних середовищах.

---

## 📂 Структура проєкту

src/backend/
│── dtos/          # Data Transfer Objects (валідація та опис вхідних даних)
│── midelwares/    # Middleware (обробка помилок, кукі, аутентифікація)
│── models/        # Моделі даних (наприклад, User)
│── router/        # Визначення маршрутів API, інтеграція Swagger
│── services/      # Логіка роботи застосунку (користувачі, авторизація, кукі)
│── index.js       # Точка входу, запуск сервера Express
│── Dockerfile     # Контейнеризація застосунку
│── package.json   # Залежності проєкту
│── package-lock.json


---

## ⚙️ Використані технології

- **Node.js** – серверне середовище
- **Express.js** – фреймворк для REST API
- **JWT** – аутентифікація користувачів
- **Swagger** – документація API
- **Docker** – контейнеризація
- **Cookie-parser** – робота з кукі
- **Mongoose/Sequelize** – робота з базою даних (залежно від реалізації моделей)
